The service

Julie Smith ("me", "I") operates the Google Tasks Import ("GTI") and Google Tasks Backup ("GTB") websites at https://import-tasks.appspot.com and https://tasks-backup.appspot.com

  • GTI allows a user to import tasks from a correctly formatted user-uploaded file to the Google Tasks server.
    • GTI suppports the GTB "Import/Export" GTBack and CSV file formats.
  • GTB allows a user to retrieve their tasks from the Google Tasks server, and view or export their tasks in a number of different formats.
    • The "Import/Export" GTBack and CSV formats are specifically designed to be used by GTI.

These services are provided free of charge, as they are running as free Google App Engine applications. Google have generously extended the free quota for GTI and GTB, so that other people can use GTI and GTB.

This privacy policy covers GTB. The privacy policy for GTI is at https://import-tasks.appspot.com/privacy

Collection and usage of data

In order to provide this service, the user is authenticated with Google, which provides GTB with the following;
  • Temporary access to be able to retrieve tasks for the user
    • Access to a user's tasks automatically expires, as specified during the Google authentication process
  • The user's name and email address
    • This is the name and email address associated with the Google account that is used when the user authenticates with Google
    • The account name and email address is stored in logs, which may be used by the system administrator to investigate user-reported problems
Once authorized by the user, GTB retrieves the user's tasks from the Google Tasks server, then encrypts and stores the tasks in a local data store to enable fast display or export of tasks.

Data security

GTB uses standard HTTPS for all communications between the browser and the GTB application.

Each user's tasks are encrypted with a user-specific key whilst they are stored in GTB, and can only be decrypted with the specific key stored in the user's browser. See "Encryption details" below for more details.

Server logs

Google App Engine automatically logs information about each request to GTB. This information includes;
  • Time of the request
  • IP address
  • User name
  • Browser version
  • Operating system

Only system administrators have access to server logs. The user's tasks, name and email address will not be shared with anyone.

Cookies

Cookies are used for four purposes in GTB;
  • Tasks are encrypted, so that they can only be accessed by you. The encryption key is stored in a cookie in your browser. If cookies are not enabled, you cannot use GTB to export or view your tasks.
  • To manage user selections such as export type, timezone and tasks display settings.
  • Google uses cookies to track your authorisation credentials between pages. This ensure that you (and only you) can get the results of your backup request.
  • Google Analytics is used to track usage such as; number of unique visitors, location, browser type, operating system.

Appendix: Encryption details

  • For each new backup job, GTB generates a random RSA key, and a random AES key
  • The RSA private key is stored as a cookie in the user's browser
    • The RSA private key is not stored by GTB
  • The user's tasks are encrypted using the AES key
  • The AES key is then encrypted with the RSA public key
  • When the user chooses a display or export option, the private RSA key is used to decrypt the AES key
  • The decrypted AES key is used to decrypt the users tasks, which are then displayed or exported
  • Without the private key, the tasks cannot be decrypted, and the user will need to start a new backup job
