This is the Django template used by tasks-backup.py to display progress to the user, and once the backup is complete, to allow the user to choose what to do with the retrieved data (export as file, email, or display as a web page). We store the user's selections in a cookie when a form is submitted. This is so that we can set the user's last chosen options when we return to this page, especially if returning after re-authenticating, but also useful when user returns from displaying tasks web page display. Note that this page uses the include_completed, include_hidden and include_deleted variables to selectively include/exclude the relevant parts of Javascript and HTML for those functions. Import/Export GTBak
 The GTBak format is designed to be used by Import Google Tasks to import tasks. Unlike the CSV format, this format retains international and extended characters.
This format is used by Google Tasks Import, which can be used to import tasks into the same or a different Google account.
Whereas the CSV format may lose international and extended characters, this format should retain all characters.
Unlike the CSV format, the GTBak format is not easily edited by the user. The file is encoded using Python pickle serialisation to ensure that no extended character information is lost or modified.
This format backs up everything except Task Links.
As far as I am aware, the only use for Task Links is when an email message is turned into a task; the link then refers to the original email message. If tasks are restored into a different account, Task Links would not work (because they would refer to messages in another account).
